


    Map



이어서 Map의 핵심 기능들을 알아보는 시간입니다. 

새로 Map을 만들어서 이번에는 키 타입을 정수타입, 벨류 타입을 스트링으로 만들어 봤습니다. 

유저의 고유 아이디 번호는 아는데 유저의 이름을 모른다고 했을때 사용할 수 있을겁니다. 

Map도 Get과 Set을 사용할 수 있기는합니다. 그래도 배열과 마찬가지로 Get에서 이어진 여러 기능을 통해 하는게 많습니다.   
Add 로 데이터를 추가하고 Remove로 지우고자 하는 키값을 넘겨주면 삭제도 됩니다. 

그리고 데이터를 Map에 추가할때 주의해야할것이 있습니다. 키값은 고유해야 한다는것인데 만약 이미 추가되있던 키값을 또 한번 Add 하게되면
벨류가 덮어 씌워집니다. 

데이터를 찾는 계열함수도 당연히 있을겁니다. Find는 키값을 받아서 데이터를 찾아주는데 반환으로 벨류와 Find가 성공했는지 불리언을 반환합니다. 

이어서 Clear는 단순하게 Map에 있는 데이터를 모두 밀어주는것입니다. 
또 IsEmpty 로 Map에 데이터가 하나라도 있는지를 알수 있습니다.
Length도 있는데 배열과 다르게 이 Map의 크기를 알아내는게 그렇게 유용하지는 않습니다. 그래도 경우에 따라 필요하다 싶으면 사용할 수 있습니다. 

좀 더 중요한것은 우리가 배열을 사용할때 모든 원소들에 적용하기 위해 쭉 순회를 돌았었습니다. 
마찬가지로 Map에서도 모든 데이터를 대상으로 어떤 작업을 하고 싶을때 Keys, Values 라는 기능들이 있습니다.
Keys 는 재밌게도 배열을 반환합니다. 키값의 배열을 반환합니다. 무슨 의미이냐면  Map에 들어있는 모든 데이터들의 키값을 배열 형태로 반환해줍니다. 
반대로 Values는 Map의 모든 Value를 배열로 반환합니다.
